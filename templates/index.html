Resumen de la Solución

Se desarrolló un pipeline de clasificación multiclase de texto. Para mejorar el desempeño se aplicó undersampling y se limitaron las clases con más de 10,000 muestras. Se entrenaron distintos modelos y se evaluaron con métricas robustas.

En un entorno de producción, se recomienda usar técnicas de oversampling como SMOTE, sin embargo, por limitaciones de hardware, no fue posible aplicarlas completamente.

Resultados

El mejor modelo alcanzó una precisión del 57%. Aunque no es viable para producción (ideal ≥ 60%), representa una buena base considerando las restricciones computacionales.

Se seleccionó Logistic Regression como modelo final.

Conclusión

La solución es modular, eficiente y escalable. Puede integrarse fácilmente en un entorno productivo gracias al uso de buenas prácticas de ingeniería de software y manejo de datos.

A futuro se sugiere aplicar Oversampling, técnicas avanzadas de Hiperparameter Tuning y evaluar modelos más complejos como redes neuronales o transformers.
